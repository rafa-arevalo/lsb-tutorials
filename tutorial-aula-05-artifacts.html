Sobre esta aula
CursoAI for Business
Duração60 minutos
TemasArtifacts · File Creation
ContextoAnálise de entrevistas e priorização de proposta de valor

Até agora todas as respostas do Claude chegaram como texto numa bolha de conversa — útil para pensar, mas difícil de usar fora do chat. Os Artifacts resolvem isso: em vez de texto, o Claude gera arquivos renderizáveis independentes — tabelas, gráficos, documentos, mini-apps interativos — que aparecem numa área separada e podem ser exportados.

Nesta aula você vai usar os Artifacts para transformar as transcrições de entrevistas em análises visuais, matrizes de priorização e comparativos de proposta de valor.

Objetivos de aprendizagem
1Entender o que são Artifacts e como diferenciá-los de resposta comum
2Gerar Artifacts de diferentes tipos — tabelas, gráficos, documentos e componentes interativos
3Iterar sobre um Artifact dentro da conversa sem recriar do zero
4Exportar Artifacts como PDF ou DOCX
01. O que é um Artifact
A diferença entre resposta e entregável
Resposta comum Artifact
Texto corrido na conversaArquivo renderizado numa área separada
Some quando você rola a telaFica fixo durante toda a conversa
Difícil de compartilharExportável como PDF, DOCX ou link
Bom para explicaçõesBom para documentos, tabelas, apps
Quando pedir um Artifact

Sempre que o resultado for algo que você vai usar fora do Claude — apresentar, entregar, imprimir ou compartilhar.

Como pedir

Use a palavra "Artifact" no prompt. Isso sinaliza ao Claude que o resultado deve ser gerado em área separada, não como texto na conversa.

Gere um Artifact com uma tabela de...
💡 Regra simples: se você vai apresentar, entregar ou exportar — peça como Artifact. Se é só para pensar — resposta normal está ótimo.
02. Tipos de Artifact e como iterar
O que você pode gerar e como refinar sem recomeçar
📊
Tabelas e matrizes
Análise de dores, comparação de concorrentes, priorização de funcionalidades
📄
Documentos Markdown
Relatórios, resumos, briefings para entrega acadêmica
⚙️
Componentes interativos
Dashboards HTML, matrizes clicáveis, checklists dinâmicos
📈
Gráficos e visualizações
Barras, dispersão, pizza — com dados das entrevistas
Como iterar sobre um Artifact
1Peça o Artifact com prompt inicial — não precisa ser perfeito
2Avalie o resultado e identifique o que precisa mudar
3Peça ajustes na mesma conversa — "adicione uma coluna de...", "mude a cor de..."
4O Claude atualiza preservando o que estava bom
💡 Dica de eficiência: comece com o mínimo necessário e refine em 2 ou 3 rodadas. É mais rápido do que tentar descrever o Artifact perfeito no primeiro prompt.

04. Demo 1 — Análise de entrevistas
De transcrições brutas a tabela de dores estruturada
Demo 1 Artifact de análise de entrevistas 12 min
Passo 1 — Analisar as 3 transcrições

Abra uma conversa dentro do Project. Envie o prompt abaixo seguido das 3 transcrições copiadas da seção anterior:

Vou colar abaixo as transcrições de 3 entrevistas com donos de clínicas odontológicas realizadas para validar o problema do OdontoFlow. Analise as transcrições e gere um Artifact no formato de tabela com as colunas: - Dor identificada - Entrevistados que mencionaram (ex: Dr. Marcos, Dra. Carla) - Frequência (quantos dos 3 mencionaram: 1/3, 2/3 ou 3/3) - Impacto percebido (alto / médio / baixo — com justificativa de 1 linha) - Citação direta (trecho exato de um entrevistado) [cole as 3 transcrições aqui]
🎙️ Fala para a turma: "Estou colando três transcrições brutas — exatamente como saem de uma entrevista real. O Claude vai extrair os dados relevantes e organizar numa tabela que posso apresentar para o professor ou usar para tomar decisões."
Passo 2 — Iterar sobre o Artifact

Após o Artifact ser gerado, envie:

Adicione uma linha de resumo ao final da tabela indicando qual é a dor mais crítica com base na combinação de frequência e impacto.
🎙️ Fala para a turma: "Notem que pedi um ajuste simples na mesma conversa — o Claude atualizou o Artifact sem recriar tudo do zero."
05. Demo 2 — Matriz de priorização interativa
Componente HTML interativo a partir das dores identificadas
Demo 2 Matriz de priorização — Artifact interativo 10 min
Passo 1 — Gerar a matriz interativa

Continue na mesma conversa da Demo 1. Envie:

Com base nas dores identificadas na tabela acima, gere um Artifact de matriz de priorização interativa (HTML) com: - Eixo Y: impacto para o negócio do dentista (baixo a alto) - Eixo X: frequência de menção nas entrevistas (baixa a alta) - Cada dor plotada como um ponto colorido com o nome identificado - Quadrantes nomeados: "Resolver primeiro" (alto impacto, alta frequência), "Monitorar" (alto impacto, baixa frequência), "Oportunidade secundária" (baixo impacto, alta frequência) e "Desprioritizar" (baixo, baixo)
Passo 2 — Adicionar legenda

Após o Artifact, envie:

Adicione uma legenda abaixo da matriz explicando o que cada quadrante significa em termos de decisão de produto.
🎙️ Fala para a turma: "Esse componente interativo pode ser apresentado numa reunião, compartilhado como link ou incorporado num relatório. E foi gerado em menos de 2 minutos a partir de transcrições brutas de entrevistas."
06. Demo 3 — Comparativo de propostas de valor
Tabela de decisão com pontuação e recomendação
Demo 3 Avaliando propostas de valor com Artifact 10 min
Passo 1 — Gerar o comparativo

Continue na mesma conversa. Envie:

Com base nas dores identificadas nas entrevistas — especialmente a dor de "falta de follow-up automático" e "dependência de comportamento humano" — sugira 3 propostas de valor distintas para o OdontoFlow. Gere um Artifact de tabela comparativa com os critérios: - Clareza para o usuário (1–5) - Diferenciação em relação a concorrentes como iClinic e OdontoSoft (1–5) - Viabilidade técnica para um estudante construir em 3 meses (1–5) - Alinhamento com as dores de maior impacto validadas (1–5) Adicione linha de pontuação total e indique a proposta recomendada com justificativa de 2 linhas.
🎙️ Fala para a turma: "Esse Artifact vira diretamente a entrega da fase de priorização do projeto. Com uma tabela assim, é muito mais fácil apresentar e justificar a decisão para o professor."
07. Demo 4 — Exportação como PDF
Consolidar tudo num relatório acadêmico exportável
Demo 4 Relatório consolidado pronto para entrega 5 min
Passo 1 — Gerar o relatório consolidado

Continue na mesma conversa. Envie:

Gere um documento Artifact em Markdown consolidando: 1. Resumo executivo das 3 entrevistas (principais achados em 5 linhas) 2. Tabela de dores com frequência e impacto 3. Proposta de valor escolhida com justificativa Formate como relatório acadêmico com título, data e seções numeradas. Título: "Relatório de Validação de Problema — OdontoFlow"
Passo 2 — Solicitar exportação em PDF

Após o Artifact, envie:

Exporte esse relatório como arquivo PDF pronto para entrega acadêmica, com o título "Relatório de Validação — OdontoFlow" e a data de hoje no cabeçalho.
🎙️ Fala para a turma: "Em menos de 15 minutos saímos de três transcrições brutas e chegamos a um relatório de validação formatado e pronto para entregar. Isso é o que significa usar Artifacts — não apenas conversar com a IA, mas produzir entregáveis reais."
08. Prática + Entregável
Aplique para o seu Micro-SaaS
Passo 1 — Artifact de análise de entrevistas

Cole as notas de uma ou duas entrevistas já realizadas. Se ainda não fez entrevistas, peça ao Claude para simular com base no relatório de dores da Aula 3.

Analise as notas de entrevista abaixo e gere um Artifact de tabela com: - Dor identificada - Frequência de menção - Impacto (alto / médio / baixo) - Citação direta [cole suas notas]
Passo 2 — Matriz de priorização
Com base na tabela de dores, gere um Artifact de matriz de priorização com impacto no eixo Y e frequência no eixo X. Plote cada dor como ponto com nome identificado. Nomeie os quadrantes: "Resolver primeiro", "Monitorar", "Oportunidade secundária" e "Desprioritizar".
Passo 3 — Comparativo de propostas de valor
Com base nas dores de maior impacto, sugira 3 propostas de valor distintas para o meu Micro-SaaS. Gere um Artifact de tabela comparativa com: - Clareza para o usuário (1–5) - Diferenciação em relação a concorrentes (1–5) - Viabilidade técnica para um estudante (1–5) - Alinhamento com dores validadas (1–5) Adicione pontuação total e indique a proposta recomendada.
Passo 4 — Exporte o Artifact principal
Exporte a análise de entrevistas como PDF com o título "Relatório de Validação — [nome do projeto]" e a data de hoje no cabeçalho.
💾 Onde salvar: salve no Google Drive na pasta do projeto — esse PDF é um entregável acadêmico.
