Nas duas primeiras aulas você aprendeu a conversar bem com o Claude e a criar uma estrutura de trabalho persistente para o seu projeto. Mas até agora, tudo ficou dentro do chat — texto numa janela de conversa.

01. O que são Artifacts
E quando pedir um

Um Artifact é qualquer output que o Claude gera como um arquivo renderizável e independente — não apenas texto numa bolha de resposta. Ele aparece numa área própria ao lado da conversa, onde você pode visualizar, editar, copiar e exportar.

Resposta comum vs. Artifact
Resposta comumArtifact
Texto corrido na conversaArquivo renderizado numa área separada
Desaparece quando você rola a telaFica fixo e acessível durante toda a conversa
Difícil de compartilhar diretamentePode ser exportado como PDF, DOCX ou copiado
Ótimo para explicações e análisesÓtimo para documentos, tabelas, apps e visualizações
Quando pedir um Artifact

Sempre que o resultado da sua conversa for algo que você vai querer usar fora do Claude — enviar para alguém, apresentar, imprimir, iterar visualmente ou reutilizar mais tarde.

02. Tipos de Artifact
O que você pode criar

O Claude consegue gerar Artifacts em vários formatos. Os mais úteis para o seu projeto:

TabelasTabelas e matrizes
Organizam dados comparativos de forma clara. Perfeito para analisar dores de entrevistas, comparar concorrentes ou priorizar funcionalidades.
MarkdownDocumentos Markdown
Textos estruturados com títulos, listas, destaques e formatação. Úteis para relatórios, resumos executivos, roteiros e briefings.
InterativoComponentes interativos (HTML/React)
Mini-aplicativos que funcionam direto no navegador — calculadoras, formulários, dashboards simples, checklists clicáveis. Não exigem nenhum conhecimento de programação para solicitar.
GráficosGráficos e visualizações
Gráficos de barra, pizza, dispersão e outros a partir de dados que você fornece ou que o Claude gera. Ideais para apresentações e relatórios.
CódigoBlocos de código
Código em qualquer linguagem, formatado e pronto para usar. Relevante para a Aula 4, quando você vai começar a construir o seu Micro-SaaS.
03. Como iterar sobre um Artifact
Refinamento progressivo sem perder o que foi feito

A grande vantagem dos Artifacts é que você pode refiná-los progressivamente dentro da mesma conversa — sem perder o que já foi feito.

O fluxo natural de refinamento
1Peça o Artifact com um prompt descritivo
2O Claude gera a versão inicial
3Você avalia e pede ajustes diretamente na conversa
4O Claude atualiza o Artifact preservando o que estava bom
Instruções de iteração para Artifacts
Exemplos prontos para usarAdicione uma coluna de prioridade (alta, média, baixa) na tabela. Mude o gráfico de barras para um gráfico de dispersão com impacto no eixo Y e frequência no eixo X. Transforme essa tabela num documento de uma página com introdução e conclusão. Adicione uma seção de recomendações ao relatório. Deixe o checklist interativo — quero poder marcar os itens como concluídos.
💡 Dica: você não precisa descrever o Artifact perfeito de cara. Comece com o mínimo necessário e vá refinando. Três rodadas de ajuste costumam produzir um resultado muito melhor do que um prompt longo e detalhado logo de início.
04. Skills
Ensinando o Claude a trabalhar do seu jeito

Uma Skill é um conjunto de instruções salvas que define como o Claude deve executar um tipo específico de tarefa. Pense nela como um template inteligente que o Claude aplica automaticamente sempre que você sinaliza que quer aquele tipo de output.

Por que criar Skills

Sem uma Skill, você precisa reescrever o mesmo prompt detalhado toda vez que quiser a mesma estrutura. Com uma Skill, você escreve uma vez — e o Claude repete o padrão sempre que você precisar.

Exemplos de Skills úteis para o seu projeto
Skill 1Análise de entrevista de cliente
Sempre que você colar notas de uma entrevista, o Claude estrutura automaticamente em: dores identificadas, frequência, impacto percebido e citação direta do entrevistado.
Skill 2Resumo de artigo acadêmico
Sempre que você colar um texto longo, o Claude entrega: problema pesquisado, metodologia, principais achados e implicações práticas.
Skill 3Feedback de pitch
Sempre que você descrever sua proposta de valor, o Claude avalia: clareza, diferenciação, evidências de problema e próximos passos sugeridos.
Como criar uma Skill
1Acesse Configurações → Skills → Nova Skill
2Escreva um nome e uma descrição de quando a Skill deve ser usada
3Escreva as instruções detalhadas de como o Claude deve executar a tarefa
4A partir daí, o Claude identifica automaticamente quando ativar essa Skill
💡 Dica: boas Skills têm um gatilho claro ("sempre que eu colar notas de entrevista...") e um output definido ("estruture em quatro seções..."). Quanto mais específica, mais consistente o resultado.

Levando seus entregáveis para o mundo real

Além de visualizar os Artifacts no Claude, você pode exportá-los como arquivos que funcionam fora da plataforma.

PDF — relatórios e documentos formais DOCX — editar e compartilhar com outros CSV/XLSX — tabelas para Excel ou Sheets
Como exportar

Basta pedir diretamente na conversa, referenciando o Artifact que você quer exportar:

Exemplos de pedidos de exportaçãoExporte esse Artifact como PDF. Gere esse documento em formato Word. Exporte essa tabela como CSV para abrir no Excel.
💡 Dica: para exportações mais caprichadas, especifique título, subtítulo e data: "Exporte como PDF com título 'Análise de Entrevistas — [projeto]' e a data de hoje no cabeçalho."

Acompanhe a tela e observe os resultados

O professor vai demonstrar ao vivo os quatro cenários desta aula. Acompanhe e observe como cada recurso funciona na prática.

1 Análise de entrevista como Artifact. Transcrição fictícia de entrevista com usuário → Artifact de tabela com dores, frequência e impacto. Como o Artifact aparece em área separada e como editá-lo com instruções adicionais.
2 Matriz de priorização interativa. A partir da tabela de dores, um Artifact de matriz (eixo impacto × eixo frequência) como componente interativo, com as dores plotadas como pontos. Ajustes de visualização com instruções simples.
3 Comparação de propostas de valor. Com base nas dores, 3 propostas de valor distintas comparadas num Artifact de tabela com critérios: clareza, diferenciação, viabilidade técnica e alinhamento com as dores.

07. Atividade Prática
Sua vez. Cada passo leva de 5 a 10 minutos.
Passo 1 — Artifact de análise de entrevistas

Cole as notas ou a transcrição de uma ou duas entrevistas que você já realizou. Se ainda não fez entrevistas, peça ao Claude para simular respostas de um usuário do seu segmento com base no relatório do Deep Research da Aula 2.

Prompt — Análise de entrevistasAnalise as notas de entrevista abaixo e gere um Artifact no formato de tabela com as seguintes colunas: - Dor identificada - Frequência (quantas vezes foi mencionada ou sugerida) - Impacto percebido pelo entrevistado (alto / médio / baixo) - Citação direta (trecho exato da fala do entrevistado) [cole suas notas aqui]
Passo 2 — Matriz de priorização

Com a tabela de dores gerada, crie um Artifact visual de priorização:

Prompt — Matriz de priorizaçãoCom base nessa tabela de dores, gere um Artifact de matriz de priorização com impacto no eixo Y e frequência no eixo X. Plote cada dor como um ponto identificado pelo nome. Destaque as dores no quadrante de alto impacto e alta frequência.

Depois de gerado, itere:

Iterações para refinar a matrizAumente o tamanho dos pontos proporcionalmente ao impacto. Adicione uma legenda explicando os quadrantes. Mude as cores para distinguir dores validadas de dores apenas mencionadas.
Passo 3 — Comparação de propostas de valor

Com base nas dores priorizadas, peça ao Claude para sugerir e comparar propostas de valor:

Prompt — Comparação de propostasCom base nas dores identificadas nas entrevistas, sugira 3 propostas de valor distintas para o meu Micro-SaaS. Para cada uma, gere um Artifact de tabela comparativa com os critérios: - Clareza para o usuário - Diferenciação em relação a concorrentes - Viabilidade técnica para um estudante - Alinhamento com as dores de maior impacto Use uma escala de 1 a 5 para cada critério e adicione uma linha de pontuação total.
Passo 4 — Crie sua primeira Skill

Acesse Configurações → Skills → Nova Skill e crie a Skill abaixo — você vai reutilizá-la ao longo do semestre:

Instruções da Skill — Análise de entrevista de clienteNome: Análise de entrevista de cliente Descrição: Use quando eu colar notas ou transcrição de uma entrevista com usuário Instruções: Sempre que eu colar notas ou transcrição de uma entrevista, estruture a análise em um Artifact com quatro seções: 1. Dores identificadas — lista das dores mencionadas, com frequência e impacto (alto/médio/baixo) 2. Comportamento atual — como o entrevistado resolve o problema hoje 3. Citações relevantes — trechos diretos que evidenciam as dores principais 4. Hipóteses levantadas — o que essa entrevista sugere sobre o problema central do projeto
Passo 5 — Exporte seu Artifact principal

Escolha o Artifact mais importante que você gerou hoje — provavelmente a tabela de análise ou a matriz de priorização — e exporte como PDF:

Prompt — Exportar como PDFExporte essa análise como um documento PDF formatado, com título "Análise de Entrevistas — [nome do seu projeto]" e a data de hoje no cabeçalho.
