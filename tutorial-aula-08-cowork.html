01. Objetivo de aprendizagem
Automatizar uma tarefa real do OdontoFlow a partir de um único CSV

Usar o Claude Cowork para automatizar uma tarefa operacional real do OdontoFlow — partindo de um único arquivo CSV de pacientes — e gerar um output que poderia ser usado sem edição pela recepcionista ou pelo dentista da clínica.

💡 Critério de sucesso: se o profissional que vai usar o output (recepcionista, dentista) precisaria editar antes de enviar, o prompt ainda não está bom — itere mais uma vez.
02. Conceito — Cowork
O Claude que opera no seu desktop — e executa o trabalho

Cowork é um Claude que opera no seu desktop. Ele tem permissão para abrir arquivos, ler conteúdo, executar tarefas e criar novos arquivos. A diferença para o Claude no chat é simples mas grande: lá você pergunta e copia a resposta; aqui ele executa o trabalho.

Essa é a resposta direta a uma das maiores demandas de quem usa IA em negócios: parar de copiar e colar entre o chat e a ferramenta de trabalho. O Cowork remove esse intermediário.

Quando faz sentidoQuando NÃO faz sentido
Tarefas repetitivas com regras clarasTarefas que exigem julgamento crítico humano
Arquivos locais (planilhas, documentos, pastas)Sistemas críticos sem revisão prévia
Output que precisa ser revisado mas não construído do zeroTarefas que mudam a cada execução
⚠️ Cuidados obrigatórios: sempre teste em pasta isolada antes de soltar em pasta de produção. Não dê acesso a arquivos com dados sensíveis sem revisar antes. Comece simples e aumente o escopo aos poucos.
📌 Insight: Cowork não substitui pessoa em decisão. Substitui pessoa em execução repetitiva. Quem manda continua sendo você — quem rola a tarefa é o Claude.
03. O que você vai ver
CSV de pacientes → mensagens de WhatsApp personalizadas, sem intervenção no meio

O professor vai usar um único arquivo CSV com 35 pacientes fictícios do OdontoFlow para mostrar uma automação completa: a partir do CSV, o Cowork vai filtrar pacientes com valor pendente, gerar uma mensagem de WhatsApp personalizada para cada um (citando nome, procedimento e valor) e salvar tudo em um arquivo de texto pronto para a recepcionista colar e enviar.

Cenário do mundo real: a recepcionista da clínica gastaria 1 a 2 horas fazendo isso na mão. O Cowork faz em minutos — e com personalização melhor do que humano cansado faria.

🗣️ Ponto central da demo: não é o Cowork "respondendo". É o Cowork lendo o CSV, filtrando os pacientes certos, gerando mensagens diferentes para cada um e salvando o arquivo — sem você intervir no meio.
04. Prompt da demonstração
O professor aponta o Cowork para a pasta do CSV e roda este comando
Demo Mensagens de cobrança via WhatsApp

O professor vai apontar o Cowork para a pasta onde está o CSV e usar o seguinte comando:

Prompt da demo — colar no CoworkLeia o arquivo base_pacientes_odontoflow.csv que está na pasta atual. Filtre apenas os pacientes que têm valor_pendente maior que zero. Para cada paciente filtrado, gere uma mensagem de WhatsApp curta para enviar amanhã. A mensagem precisa: - Cumprimentar pelo primeiro nome do paciente - Citar o procedimento que foi feito (último_procedimento) - Mencionar o valor pendente de forma educada (não rude) - Oferecer opção de parcelamento em até 3x sem juros - Encerrar com frase amigável e profissional - Ter no máximo 4 linhas O tom precisa ser de clínica que se importa com o paciente, não de cobrador. Cada mensagem precisa ser personalizada — nada de texto-padrão idêntico para todo mundo. Salve todas as mensagens em um arquivo chamado mensagens_cobranca.txt, separando cada mensagem com uma linha em branco e identificando cada uma com o nome completo do paciente como cabeçalho.
05. O que observar
3 sinais visíveis de que o Cowork está executando — não simulando

Três coisas que ficam visíveis quando o Cowork está executando:

1Leitura real do arquivo. O Cowork mostra que abriu o CSV, conta quantas linhas leu, identifica as colunas. Não é simulação.
2Filtragem correta. Só os pacientes com valor pendente entram no output. Os outros são ignorados — sem você precisar pré-processar nada.
3Personalização real. As mensagens não são copy-paste com o nome trocado. Cada uma reflete o procedimento específico daquele paciente. Texto genérico copiado para todos seria fácil de fazer no Excel — Cowork entrega algo que o Excel não consegue.
06. O que você vai fazer
Mesmo CSV da demo — tarefa diferente, em duplas

Você vai usar o mesmo arquivo CSV que o professor usou na demonstração (já compartilhado com a turma) para automatizar uma tarefa diferente do OdontoFlow.

Passos
1Baixe o arquivo (1 min) — base_pacientes_odontoflow.csv da sala virtual. Salve em pasta acessível para o Cowork.
2Escolha sua tarefa (1 min) — em duplas, cada dupla escolhe uma diferente entre A, B, C ou D abaixo.
3Escreva o prompt (4 min) — use o molde da próxima seção e adapte.
4Execute e revise (5 min) — aponte o Cowork para a pasta e rode. Avalie: o output está usável sem edição?
5Itere se necessário (1 min) — se saiu genérico ou com erro, ajuste o prompt e rode de novo.
As 4 tarefas — escolha uma
TarefaO que filtrarO que gerar
A — AniversariantesPacientes que aniversariam neste mêsMensagem personalizada usando o último procedimento como gancho de retorno à clínica
B — ReativaçãoStatus inativo OU última consulta há mais de 6 mesesMensagem de reativação personalizada por procedimento
C — Follow-upLimpeza, canal ou extração nas últimas 4 semanasMensagem de follow-up apropriada para cada procedimento

07. Prompt para você usar
Molde para adaptar à sua tarefa escolhida (A, B, C ou D)

Adapte este molde para a sua tarefa escolhida:

Molde de prompt — adaptar à sua tarefaLeia o arquivo base_pacientes_odontoflow.csv que está na pasta atual. FILTRO QUE EU PRECISO [Descreva qual subgrupo de pacientes interessa para a sua tarefa. Exemplos: - "Pacientes que aniversariam no mês corrente" - "Pacientes com status inativo OU última consulta há mais de 180 dias" - "Pacientes cujo último procedimento foi limpeza, canal ou extração E última consulta foi nos últimos 28 dias" - "Todos os pacientes — quero análise consolidada"] O QUE VOCÊ DEVE GERAR [Descreva o output desejado. Exemplos: - "Uma mensagem de WhatsApp personalizada por paciente, com no máximo 4 linhas, citando [campos relevantes]" - "Um relatório resumido em texto estruturado, com números reais extraídos do CSV"] REGRAS DE TOM [Profissional / amigável / direto / informal] - O que evitar: [frases vazias, tom de cobrança, urgência artificial, etc.] - Personalização real é obrigatória: nada de texto-padrão idêntico com nome trocado. ARQUIVO DE SAÍDA Salve em [nome_arquivo.txt ou nome_arquivo.csv], separando cada item com [...].
💡 Critério de qualidade: o output saiu usável de verdade ou ficou genérico? Se a recepcionista ou o dentista da clínica precisaria editar antes de usar, o prompt ainda não está bom. Itere mais uma vez.
08. O que fazer com o output
3 perguntas que validam o uso real da automação

O arquivo gerado pelo Cowork é o que mostra que a automação funciona — não a tela do Cowork rodando.

1Abra o arquivo de saída. Leia pelo menos 3 itens (3 mensagens, 3 linhas do relatório) e avalie se você usaria sem editar.
2Estime o tempo economizado. Se essa tarefa rodasse na clínica de verdade toda semana, quanto tempo de recepcionista ela liberaria por mês? Esse cálculo é o que justifica o investimento em IA para qualquer negócio.
3Pense em outras tarefas que esse mesmo CSV resolveria. "O que mais esse arquivo me permite automatizar?" é o pulo do gato. Dado bom é dado reutilizável. Você não precisa de cinco planilhas para cinco automações; precisa de uma planilha bem estruturada.
⚠️ Importante: antes de rodar Cowork em arquivos do seu negócio real, sempre teste em pasta isolada com dados fictícios primeiro. Cowork mexe em arquivos de verdade, e arquivo deletado por engano não volta. A regra é: testou, funcionou, então parta para os dados reais — nunca o contrário.
