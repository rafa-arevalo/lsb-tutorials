Segunda atividade prática da aula final. Você vai aprender a extrair um design system de um site de referência usando a visão do Claude e depois reaproveitar esse design system para gerar telas novas do OdontoFlow em formato de artefato HTML navegável — sem precisar de designer.

Objetivos de aprendizagem
1Usar a visão do Claude para descrever cores, tipografia e estilo a partir de prints
2Gerar um design system documentado que pode ser colado em qualquer prompt futuro
3Aplicar esse design system para criar uma tela do OdontoFlow em HTML respeitando a identidade visual
4Aprender a iterar sobre artefatos HTML em segundos quando algo precisa de ajuste
01. Objetivo de aprendizagem
Do print do site de referência à tela do seu produto, passando pelo design system

Pegar prints de um site de referência profissional, usar o Claude para extrair o design system completo daquele site (cores, tipografia, espaçamentos, estilo de botões e cards) e em seguida gerar uma tela do OdontoFlow em artefato HTML que respeite esse design system.

💡 Critério de sucesso: a tela gerada precisa parecer que poderia estar no site de referência — mesma família visual, sem ser cópia. Se ficou genérica ou destoante, o problema está no design system extraído ou na descrição da tela — não no Claude.
02. Conceito — Design System extraído
O que faz um site parecer coerente — e como o Claude extrai isso de imagens

Quando você vê um site bem feito de uma empresa que admira, sente que algo é bonito mas não consegue traduzir em palavras o que faz aquele site funcionar. Isso acontece porque o que está por trás é um design system — um manual visual com cores, tipografia, espaçamento, estilo de botões e cards que toda tela do produto obedece. É o que faz a marca parecer coerente.

A boa notícia: você não precisa criar um design system do zero. O Claude tem visão. Ele consegue olhar imagens e descrever em palavras o que está vendo. Isso permite uma manobra poderosa:

1Você escolhe um site cujo visual admira
2Tira prints dele
3Pede ao Claude para extrair o design system a partir dos prints
4Ganha um documento estruturado com cores, fontes e estilos
5Usa esse documento em qualquer prompt futuro para gerar telas no mesmo padrão
📌 Insight: design system é um arquivo de texto. Você cria uma vez, salva, e cola em qualquer prompt futuro para que o Claude gere qualquer tela do seu produto respeitando o mesmo padrão visual. É inspiração estruturada, não cópia.
03. O que você vai ver
Duas etapas ao vivo: extração do design system + aplicação na tela do OdontoFlow

O professor vai fazer duas etapas ao vivo:

Etapa 1 — Extração
O professor sobe 3 a 4 prints de um site de referência (escolhido antes da aula — algo limpo e profissional como Linear, Notion, Calendly, Stripe ou similar) e pede ao Claude para extrair o design system completo.
Etapa 2 — Aplicação
O professor cola o design system extraído no início de um novo prompt e pede para gerar a tela de dashboard principal do OdontoFlow como artefato HTML, respeitando o design system.

Você vai ver duas coisas que parecem mágica mas não são: o Claude descrevendo cores e tipografia que ele só viu em imagem; e a tela do OdontoFlow nascendo com a cara do site de referência sem ser cópia.

🗣️ Ponto central da demo: a tela do OdontoFlow gerada na Etapa 2 deve parecer que poderia estar no site de referência — mesma família visual, sem ser cópia. Repare como o design system documentado virou o "tradutor" entre os dois.
04. Prompts da demonstração
Etapa 1 (extração) com prints anexados · Etapa 2 (aplicação) colando o design system
Etapa 1 Extração do design system a partir dos prints

Anexar 3 a 4 prints do site de referência e colar este prompt:

Etapa 1 — anexar prints + colar este promptEstou analisando esses prints do site [NOME DO SITE DE REFERÊNCIA] como inspiração visual. Quero que você extraia um design system completo a partir do que está vendo nessas imagens. Inclua: PALETA DE CORES - Cor primária (com código hex aproximado) - Cor secundária (hex) - Cores neutras de background e texto (hex) - Cores de estado (sucesso, erro, alerta) se identificáveis TIPOGRAFIA - Família de fonte para títulos - Família de fonte para corpo de texto - Tamanhos aproximados para H1, H2, H3 e corpo - Peso (regular, medium, bold) por uso ESPAÇAMENTO - Espaçamento base (em pixels) que parece ser o padrão - Padding interno típico de cards e seções ESTILO DE COMPONENTES - Botão primário (cor, raio de borda, padding) - Botão secundário (idem) - Cards (sombra, raio de borda, padding interno) - Inputs (borda, padding, foco) SENSAÇÃO GERAL - Em 2 a 3 frases: que sensação esse design transmite (confiável, jovem, técnico, premium, etc.) Entregue como documento estruturado em markdown que eu possa colar em qualquer prompt futuro.
Etapa 2 Aplicação do design system na tela do OdontoFlow

Em uma nova conversa, colar o design system gerado na Etapa 1 no lugar indicado e pedir a tela:

Etapa 2 — colar design system + pedir a tela[COLAR AQUI O DESIGN SYSTEM EXTRAÍDO NA ETAPA 1] --- Usando rigorosamente o design system acima, crie um artefato HTML completo da tela de dashboard principal do OdontoFlow, MicroSaaS de atendimento odontológico para clínicas. A tela precisa ter: - Header com logo "OdontoFlow" e navegação (Agenda, Pacientes, Financeiro, Configurações) - Seção "Agendamentos de hoje" com 4 cards horizontais (nome do paciente, horário, tipo de procedimento) - Gráfico simples de ocupação semanal (barras) - Lista de "Mensagens não lidas do WhatsApp" com 3 itens (nome do paciente + preview da mensagem) - Botão grande de "Novo agendamento" em destaque Respeite as cores, tipografia, espaçamento e estilo de componentes do design system. A tela precisa parecer da mesma família visual do site que serviu de referência.
05. O que observar
3 coisas para reparar enquanto a tela é gerada ao vivo

Três coisas para reparar enquanto a tela é gerada:

1Visão funciona. O Claude descreveu cores e fontes que ele só viu em imagem — não estavam escritas em lugar nenhum.
2Reaproveitamento. O design system virou um arquivo. Esse arquivo pode ser colado em quantos prompts quiser, e toda tela vai sair coerente.
3Iteração rápida. Se algo não ficar bom (uma cor errada, um espaçamento esquisito), basta ajustar o design system ou pedir uma iteração — em segundos a tela se atualiza.
06. O que você vai fazer
Mesmo design system da demo — tela diferente, em duplas

Você vai usar o mesmo design system que foi extraído na demonstração (o professor compartilhou o arquivo com a turma) para gerar uma tela diferente do OdontoFlow em artefato HTML.

Passos
1Pegue o design system compartilhado (1 min) — o professor já distribuiu o arquivo na sala virtual. Abra e copie o conteúdo inteiro.
2Escolha sua tela (1 min) — em duplas. Cada dupla pega uma tela diferente para evitar repetição (ver lista abaixo).
3Monte o prompt (3 min) — cole o design system no início e descreva a sua tela com riqueza de detalhes (elementos, hierarquia, ações principais).
4Gere e itere (6 min) — avalie o artefato HTML que sair. Itere pelo menos uma vez para melhorar (ajustar layout, corrigir cores, adicionar elemento que faltou).
5Compare com a tela da demo (1 min) — coloque a sua tela ao lado do dashboard principal da demonstração. As duas precisam parecer do mesmo produto.

07. Prompt para você usar
Cole o design system no início e adapte a descrição para a sua tela

Cole o design system compartilhado no lugar indicado e adapte a descrição para a sua tela:

Molde de prompt — adaptar à sua tela escolhida[COLAR AQUI O DESIGN SYSTEM COMPARTILHADO PELO PROFESSOR] --- Usando rigorosamente o design system acima, crie um artefato HTML completo da [NOME DA SUA TELA] do OdontoFlow, MicroSaaS de atendimento odontológico para clínicas. Contexto da tela: - Quem usa: [DENTISTA / RECEPCIONISTA / PACIENTE] - Para que serve: [OBJETIVO PRINCIPAL DA TELA] - Ações principais que o usuário precisa fazer aqui: [LISTAR 2 OU 3 AÇÕES] Elementos que precisam aparecer na tela: - [ELEMENTO 1] - [ELEMENTO 2] - [ELEMENTO 3] - [ELEMENTO 4] A tela precisa respeitar cores, tipografia, espaçamento e estilo de componentes do design system acima — e parecer da mesma família visual da tela de dashboard gerada na demonstração da aula. Pelo menos um elemento da tela deve ser clicável (botão ou link funcional).
💡 Dica: descrição vaga gera tela genérica. Quanto mais você descrever quem é o usuário e qual a dor que a tela resolve, melhor o output. Pense menos em "como a tela deve ficar" e mais em "o que o usuário precisa fazer aqui".
08. O que fazer com o output
A tela que você gerou não é desenho de papel — é código rodando

A tela que você gerou não é desenho de papel. É código rodando.

1Salve o artefato HTML. Você pode usar essa tela para validar com cliente real, mostrar para sócio, levar para reunião de produto. Funciona como mockup navegável.
2Anote o que ajustaria no design system. Se você precisou "lutar" com o design system para conseguir alguma coisa específica da sua tela, isso é insumo para refinar o sistema na próxima vez.
3Guarde o design system para o projeto final. Esse arquivo (design system extraído) é insumo direto do seu projeto final do semestre. Vai usar de novo.
⚠️ Importante: o design system não é só decorativo. É o que vai permitir que qualquer tela do seu projeto final pareça profissional. Sem ele, cada tela vira uma decisão nova e o produto fica visualmente caótico.
