TemaSkills
ContextoPadronização de fluxos recorrentes do Micro-SaaS

Você já percebeu que certas tarefas do projeto se repetem toda semana: analisar entrevistas, avaliar propostas de valor, documentar o progresso. Para cada uma você reescreveu um prompt longo — e vai continuar reescrevendo até o final do semestre.

As Skills eliminam esse retrabalho. Uma Skill é um conjunto de instruções salvas que ensina ao Claude como executar uma tarefa específica. Você escreve uma vez. O Claude reproduz o padrão toda vez, sem variações indesejadas.

Objetivos de aprendizagem
1Entender o que é uma Skill e quando faz sentido criar uma
2Escrever instruções com gatilho claro e output definido
3Criar pelo menos 3 Skills aplicadas ao projeto Micro-SaaS
4Testar, identificar problemas e ajustar uma Skill
5Pedir ao Claude para criar ou melhorar uma Skill a partir de um prompt existente
01. O que é uma Skill
A diferença entre prompt e fluxo padronizado
Sem SkillCom Skill
Reescreve o mesmo prompt toda vezEscreve uma vez, reutiliza sempre
Resultados inconsistentesOutput padronizado toda vez
Prompt se perde no históricoSkill salva em Configurações
Para tarefas pontuaisPara tarefas recorrentes
💡 Regra prática: se você já usou o mesmo prompt mais de duas vezes, vale transformá-lo numa Skill.

Skills não substituem bons prompts — elas preservam bons prompts e os tornam disponíveis em qualquer conversa, a qualquer momento, sem precisar lembrar como você escreveu da última vez.

02. Anatomia e como criar
Os três campos que definem uma Skill

Toda Skill tem três componentes. A qualidade de cada um determina se a Skill vai funcionar ou não.

1. Nome

Claro e descritivo. Você vai precisar reconhecer para o que serve olhando só o nome.

Fraco
  • "Análise"
  • "Avaliação"
Forte
  • "Análise de entrevista de cliente — OdontoFlow"
  • "Avaliação de proposta de valor"
2. Gatilho (Descrição)

Define quando ativar. Começa com "Use quando...".

Fraco
  • "Use quando eu pedir uma análise."
Forte
  • "Use quando eu colar notas ou transcrição de uma entrevista com dentista."
3. Instruções de output

O que produzir, em quais seções, com qual formato.

Fraco
  • "Analise e dê um resumo."
Forte
  • "Estruture em 4 seções: dores identificadas, comportamento atual, citações diretas, hipóteses levantadas."
Como criar
APelo painel: Configurações → Skills → Nova Skill
BPedindo ao Claude: cole um prompt que já funciona e peça para transformar em Skill — a Demo 3 mostra isso ao vivo

Demo 1 Criar e testar a Skill de análise de entrevista 12 min
Passo 1 — Acesse o painel de Skills

Navegue até Configurações → Skills → Nova Skill. Preencha os três campos:

Nome
Análise de entrevista de cliente — OdontoFlow
Descrição (Gatilho)
Use quando eu colar notas ou transcrição de uma entrevista com dono de clínica odontológica para o projeto OdontoFlow.
Instruções
Ao receber notas ou transcrição de entrevista, gere um Artifact estruturado com as seguintes seções: 1. DORES IDENTIFICADAS - Nome da dor (em negrito) - Frequência: 1 vez, repetidamente ou com ênfase - Impacto: alto / médio / baixo - Evidência: o que no texto indica esse impacto 2. COMPORTAMENTO ATUAL - Como resolve (ou tenta) o problema hoje - Ferramentas, processos ou gambiarras mencionadas - Por que a solução atual falha ou é insuficiente 3. CITAÇÕES DIRETAS - Mínimo 3, máximo 5 trechos exatos - Selecione os que melhor evidenciam as dores principais 4. HIPÓTESES PARA O PROJETO - O que confirma sobre o problema central do OdontoFlow - O que contradiz ou nuança a hipótese atual - Uma pergunta que essa entrevista levanta Finalize com: "Essa entrevista [confirma/contradiz/nuança] a hipótese principal porque [razão em 1 frase]."

Tornando visível a diferença de consistência
Demo 2 Comparação direta — mesmo input, resultados diferentes 8 min
Passo 1 — Sem Skill (conversa comum)

Abra uma conversa fora do Project, sem nenhuma Skill configurada. Envie a mesma transcrição do Dr. Marcos sem nenhuma instrução.

A resposta será texto corrido, sem as 4 seções e provavelmente sem Artifact.

Passo 2 — Com Skill (dentro do Project)

Volte para uma conversa dentro do Project com a Skill ativada. Envie o mesmo input.

Mostre os dois resultados lado a lado.

🗣️ Nota do instrutor: "Mesma transcrição, dois resultados completamente diferentes. A Skill garante que toda análise de entrevista — independente de quando você fizer ou em qual conversa — vai seguir o mesmo padrão. Isso é especialmente útil quando você tem 5, 8, 10 entrevistas para analisar."
06. Demo 3 — Claude cria a Skill
Transformar um prompt existente em Skill em 2 minutos
Demo 3 De prompt recorrente a Skill estruturada 8 min
Passo 1 — Peça ao Claude para criar a Skill

Abra uma conversa dentro do Project. Envie:

Tenho usado este prompt repetidamente para avaliar a proposta de valor do OdontoFlow: "Avalie esta proposta de valor considerando: (1) clareza para o usuário de 1 a 5, (2) diferenciação em relação a OdontoSoft e iClinic de 1 a 5, (3) alinhamento com a dor de follow-up manual de 1 a 5, (4) viabilidade para um estudante construir em 3 meses de 1 a 5. Dê nota, justificativa de 1 linha para cada critério, e reescreva a proposta incorporando as melhorias sugeridas." Transforme esse prompt em uma Skill estruturada com: - Nome claro e descritivo - Gatilho específico de quando deve ser ativada - Instruções de output com seções definidas Formate pronto para colar no painel de Skills do Claude.
Passo 2 — Cole no painel e teste

Copie o resultado gerado → cole em Configurações → Skills → Nova Skill. Depois teste enviando a proposta de valor do OdontoFlow (disponível na seção "Dados da Demo").

🗣️ Nota do instrutor: "Transformei um prompt que já usava em uma Skill em menos de 2 minutos. Daqui pra frente, toda vez que reformular a proposta de valor — o que vai acontecer várias vezes ao longo do semestre — recebo a avaliação com a mesma estrutura."
07. Demo 4 — Testar e ajustar uma Skill
A primeira versão raramente é perfeita — o ciclo de melhoria
Demo 4 Ciclo: criar → testar → identificar → corrigir 7 min
Passo 1 — Criar a Skill de relatório semanal (antes da aula)

Crie essa Skill antes da aula ou ao vivo, colando as instruções abaixo:

instruções da Skill de relatório semanalNome: Relatório semanal — OdontoFlow Gatilho: Use quando eu descrever o que aconteceu durante a semana no projeto. Instruções: Gere um Artifact de relatório com: - Resumo executivo (3 linhas) - O que foi feito - Decisões tomadas - Próximos passos priorizados - Bloqueios
Passo 2 — Teste com a atualização semanal

Abra uma conversa no Project e envie o Material 3 da seção "Dados da Demo" (atualização semanal sintética) sem nenhuma instrução adicional.

Passo 3 — Identifique um problema no output

Por exemplo: os próximos passos foram listados sem prazo nem dependências. Use o prompt abaixo para corrigir:

A Skill de relatório semanal gerou esse output: [cole o output] O problema: os próximos passos não têm prazo nem indicação de dependências. Reescreva as instruções da Skill para que os próximos passos sempre incluam: (1) descrição da tarefa, (2) prazo estimado e (3) dependências (se existirem). Mantenha o restante da estrutura.
Passo 4 — Cole as instruções atualizadas e teste novamente

Substitua as instruções da Skill no painel e envie o mesmo input. A saída deve incluir prazo e dependências nos próximos passos.

🗣️ Nota do instrutor: "O ciclo é: criar → testar → identificar o problema → corrigir com ajuda do Claude → testar de novo. Skills raramente ficam perfeitas na primeira versão — mas melhoram rápido."
08. Prática + Entregável
Crie as 3 Skills do seu projeto agora
Skill 1 — Análise de entrevista

Acesse Configurações → Skills → Nova Skill. Cole e adapte para o seu projeto:

Skill 1 — análise de entrevista (adaptar para seu projeto)Nome: Análise de entrevista de cliente — [nome do projeto] Gatilho: Use quando eu colar notas ou transcrição de uma entrevista com usuário potencial do meu Micro-SaaS. Instruções: Gere um Artifact com 4 seções: 1. DORES IDENTIFICADAS - Nome da dor, frequência, impacto (alto/médio/baixo), evidência 2. COMPORTAMENTO ATUAL - Como resolve hoje, ferramentas usadas, por que falha 3. CITAÇÕES DIRETAS - 3 a 5 trechos exatos que evidenciam as dores principais 4. HIPÓTESES PARA O PROJETO - O que confirma, contradiz ou nuança a hipótese central - Uma pergunta que vale investigar nas próximas entrevistas Finalize com: "Essa entrevista [confirma/contradiz/nuança] a hipótese principal porque [razão]."
Skill 2 — Avaliação de proposta de valor
Skill 2 — avaliação de proposta de valorNome: Avaliação de proposta de valor — [nome do projeto] Gatilho: Use quando eu descrever ou pedir feedback sobre a proposta de valor. Instruções: Avalie em Artifact com: 1. Notas por critério (1–5 com justificativa de 1 linha): - Clareza para o usuário - Diferenciação em relação a concorrentes - Alinhamento com as dores validadas - Viabilidade técnica para um estudante 2. Pontos fortes (até 3) 3. Gaps críticos (até 3) 4. Sugestão de reformulação em até 2 frases
Skill 3 — A partir de um prompt seu

Pense num prompt que você já usou mais de uma vez. Cole e envie:

Tenho usado este prompt com frequência: [cole seu prompt] Transforme em Skill com nome, gatilho e instruções detalhadas. Formate pronto para colar no painel de Skills.
Testar e ajustar

Para qualquer Skill que não ficou satisfatória:

Esta é minha Skill: [instruções] Testei com: [input] O output foi: [output] Problema: [o que ficou errado] Reescreva as instruções para corrigir, mantendo o que já funcionava.
