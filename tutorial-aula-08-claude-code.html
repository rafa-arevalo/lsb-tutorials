Esta é a última aula do curso — e a que vai transformar o projeto de um documento em produto. Você passou pelas etapas de pesquisa, validação, priorização e planejamento. Agora é hora de construir.

O Claude Code é uma ferramenta de linha de comando que torna o desenvolvimento acessível mesmo para quem tem pouca experiência com programação. Você descreve o que quer em linguagem natural — no terminal — e o Claude lê os arquivos, escreve o código, executa comandos e gerencia o projeto de forma incremental.

Objetivos de aprendizagem
1Instalar e configurar o Claude Code no computador
2Criar o arquivo CLAUDE.md com o contexto do projeto
3Descrever em linguagem natural e gerar o scaffold inicial

5Montar um plano de desenvolvimento para as semanas seguintes
01. O que é o Claude Code
Um agente de desenvolvimento no seu terminal

O Claude Code roda no terminal e age como um agente de desenvolvimento. Você descreve o que quer construir e o Claude:

→Lê e entende o código existente na pasta
→Escreve novos arquivos e funções
→Executa comandos de terminal (instalar dependências, rodar servidor, testes)
→Corrige bugs a partir de uma descrição em linguagem natural
→Explica o que fez e por quê
Instalação

Pré-requisito: Node.js instalado. Verifique com:

node --version

Se não aparecer versão, instale em nodejs.org (versão LTS). Depois instale o Claude Code:

npm install -g @anthropic-ai/claude-code

Para iniciar uma sessão na pasta do projeto:

cd /caminho/para/pasta/do/projeto claude
⚠️ Atenção: o Claude Code requer plano Pro ou Max. Se estiver no plano gratuito, use o Chat com Artifacts de código como alternativa para gerar e iterar sobre o código.
02. CLAUDE.md e Stack
O briefing do agente de desenvolvimento

O CLAUDE.md fica na raiz do repositório e define o contexto que o Claude Code usa em todas as sessões — equivalente às instruções do Project, mas para o ambiente de desenvolvimento. Quanto mais completo, mais específicas e consistentes as respostas do Claude.

03. Preparação
CLAUDE.md completo do OdontoFlow — pronto para usar
📋 Nota para o professor: verifique a instalação do Node.js e Claude Code antes da aula. Crie a pasta odontoflow-app/ com o CLAUDE.md abaixo. Não crie mais nenhum arquivo — o scaffold será gerado ao vivo.
Prep Verificar instalação + criar pasta
verificar antes da aula
node --version # deve mostrar v18 ou superior npm --version # deve mostrar versão instalada claude --version # deve mostrar versão do Claude Code
criar pasta da demonstração
mkdir odontoflow-app cd odontoflow-app
Prep CLAUDE.md completo do OdontoFlow — salvar na raiz da pasta
CLAUDE.md
CLAUDE.md — salvar na raiz de odontoflow-app/# OdontoFlow — Contexto para o Claude Code ## O que é o projeto OdontoFlow é um Micro-SaaS para clínicas odontológicas de pequeno porte (1 a 3 dentistas) que automatiza o follow-up com pacientes após consultas. ## Problema que resolve Clínicas perdem 40-60% dos pacientes por falta de contato sistemático pós-consulta. A comunicação hoje é manual via WhatsApp e falha por volume e turnover de equipe. ## Funcionalidade core do MVP 1. Cadastro de pacientes (nome, telefone, data da última consulta) 2. Identificação automática de pacientes sem retorno há mais de 90 dias 3. Geração de mensagem personalizada de reengajamento para cada paciente 4. Registro de status do contato (enviado, respondeu, agendou) ## Usuário principal Dono de clínica odontológica, sem equipe de TI, que precisa de uma ferramenta simples que funcione sem treinamento. ## Stack tecnológica - Backend: Python com FastAPI - Banco de dados: SQLite (arquivo local, sem configuração) - Frontend: HTML + CSS + JavaScript simples (sem framework) - Porta padrão: 8000 ## Estrutura de pastas odontoflow-app/ ├── main.py # arquivo principal da API ├── database.py # configuração e modelos do banco ├── models.py # modelos Pydantic ├── routes/ │ ├── pacientes.py # rotas de CRUD de pacientes │ └── follow_up.py # rotas de follow-up ├── static/ │ └── index.html # interface simples ├── requirements.txt # dependências Python ├── CLAUDE.md # este arquivo └── README.md # instruções de instalação ## Convenções de código - Nomes de variáveis e funções em snake_case em português - Comentários em português - Commits no formato: "feat: [descrição]" ou "fix: [descrição]" - Rotas da API: /api/v1/[recurso] - Respostas JSON sempre com campos: success, data, message ## Critérios de "pronto" - Rota funciona localmente (curl ou browser) - Retorna JSON no formato padrão (success, data, message) - Sem erros no terminal ao iniciar o servidor - README atualizado com instrução de uso da nova rota ## O que NÃO faz parte do MVP - Autenticação de usuários - Envio real de WhatsApp (apenas gerar o texto da mensagem) - Relatórios e dashboards - Múltiplas clínicas

05. Demo 2 — Scaffold inicial
De pasta vazia a servidor funcional
Demo 2 Gerando a estrutura completa do projeto 12 min
Passo 1 — Cole a instrução no Claude Code
instrução — cole no terminal do Claude CodeCrie o scaffold completo do OdontoFlow conforme definido no CLAUDE.md. Preciso que você: 1. Crie o requirements.txt com as dependências necessárias (fastapi, uvicorn, sqlalchemy, pydantic) 2. Crie o database.py com a configuração do SQLite e a tabela de pacientes com os campos: id, nome, telefone, email, data_ultima_consulta, data_cadastro 3. Crie o models.py com os modelos Pydantic para criação e resposta de paciente 4. Crie o main.py com a configuração do FastAPI e uma rota GET /api/v1/health que retorna {"success": true, "message": "OdontoFlow funcionando"} 5. Crie o README.md com instruções de instalação e como rodar o projeto Após criar os arquivos, instale as dependências e verifique se o servidor inicia sem erros.
Passo 2 — Aguarde e mostre os arquivos sendo criados

Mostre cada arquivo sendo criado no terminal. Quando o Claude terminar, suba o servidor:

uvicorn main:app --reload

Abra http://localhost:8000/api/v1/health no navegador.

🎙️ Fala para a turma: "Em menos de 5 minutos, saímos de uma pasta vazia para um servidor FastAPI funcional. Em nenhum momento eu escrevi uma linha de código — apenas descrevi o que queria."
06. Demo 3 — CRUD de pacientes
Primeira funcionalidade real em desenvolvimento incremental
Demo 3 Rotas de cadastro e listagem de pacientes 12 min
Passo 1 — Cole a instrução no Claude Code
instrução — cole no terminal do Claude CodeCrie o arquivo routes/pacientes.py com as seguintes rotas: POST /api/v1/pacientes - Recebe: nome, telefone, email (opcional), data_ultima_consulta - Valida que nome e telefone são obrigatórios - Verifica se já existe paciente com o mesmo telefone - Salva no banco e retorna o paciente criado - Formato: {"success": true, "data": {paciente}, "message": "Paciente cadastrado"} GET /api/v1/pacientes - Lista todos os pacientes cadastrados - Ordena por nome alfabeticamente - Retorna: {"success": true, "data": [lista], "message": "X pacientes encontrados"} GET /api/v1/pacientes/{id} - Busca paciente por ID - Retorna 404 se não encontrar Depois de criar o arquivo, registre as rotas no main.py e teste se o servidor reinicia sem erros.
Passo 2 — Teste com curl após o Claude terminar
curl -X POST http://localhost:8000/api/v1/pacientes \ -H "Content-Type: application/json" \ -d '{"nome": "Maria Silva", "telefone": "11999887766", "data_ultima_consulta": "2024-06-15"}'
🎙️ Fala para a turma: "O Claude escreveu a rota, registrou no servidor e testou. Notem que em nenhum momento eu escrevi uma linha de código — apenas descrevi o que queria."
07. Demo 4 — Follow-up core
A funcionalidade principal do OdontoFlow funcionando
Demo 4 Identificar pacientes para follow-up + mensagem personalizada 10 min
Passo 1 — Popule o banco com dados de teste
instrução — cole no Claude CodeCrie um script Python chamado seed.py que insere 5 pacientes de teste com as seguintes datas de última consulta: - Ana Costa: última consulta há 120 dias - Bruno Lima: última consulta há 45 dias - Claudia Rocha: última consulta há 200 dias - Diego Alves: última consulta há 15 dias - Elena Souza: última consulta há 95 dias Execute o script para popular o banco.
Passo 2 — Crie a rota de follow-up
instrução — cole no Claude CodeCrie o arquivo routes/follow_up.py com a seguinte rota: GET /api/v1/follow-up/pendentes - Retorna todos os pacientes sem retorno há mais de 90 dias - Para cada paciente, calcule e inclua: dias_sem_retorno - Ordene do mais tempo sem retorno para o mais recente - Inclua uma mensagem personalizada para cada paciente: "Olá [nome]! Faz [X] dias desde sua última consulta na nossa clínica. Que tal agendar uma revisão? Estamos à disposição! 😊" Registre a rota no main.py e teste.
Passo 3 — Teste no navegador

Abra http://localhost:8000/api/v1/follow-up/pendentes. Devem aparecer Ana Costa (120 dias), Claudia Rocha (200 dias) e Elena Souza (95 dias) — nessa ordem por dias sem retorno.

🎙️ Fala para a turma: "Essa é a funcionalidade principal do OdontoFlow. O dono da clínica abre essa rota, vê quem precisa de contato e copia a mensagem para enviar no WhatsApp. O problema que validamos nas entrevistas está resolvido."
