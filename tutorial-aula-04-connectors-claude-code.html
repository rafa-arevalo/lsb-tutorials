Sobre esta aula
CursoAI for Business
Duração60 minutos
TemasCowork · Connectors · Claude Code
ProjetoMicro-SaaS — protótipo funcional e automações

Nas três primeiras aulas você construiu a base: aprendeu a fazer bons prompts, criou um Project persistente para o seu Micro-SaaS, gerou análises de entrevistas e definiu sua proposta de valor. Agora é hora de colocar tudo isso em movimento.

Esta é a aula mais prática do curso — e também a que mais vai mudar a forma como você vê o que é possível fazer com IA. Você vai aprender a conectar o Claude às ferramentas que já usa no dia a dia via Connectors, a delegar tarefas autônomas com o Cowork, e a dar os primeiros passos com o Claude Code para gerar o scaffold inicial do seu Micro-SaaS em linguagem natural, sem precisar escrever uma linha de código do zero.

O objetivo não é que você saia desta aula com um produto pronto. É que você saia sabendo exatamente como o Claude pode acelerar cada etapa da construção — e com o primeiro pedaço do seu Micro-SaaS já funcionando.

Objetivos de aprendizagem
1Conectar o Claude a ferramentas externas como Gmail, Google Drive e Notion via Connectors
2Usar o Cowork para delegar tarefas autônomas que envolvem arquivos locais
3Entender o fluxo de trabalho do Claude Code — como instalar, configurar e usar no terminal
4Descrever em linguagem natural a funcionalidade principal do seu Micro-SaaS e gerar o scaffold inicial
5Montar um plano concreto de como usar o Claude nas próximas etapas do semestre
01. Connectors
O Claude conectado às suas ferramentas

Os Connectors são integrações que permitem ao Claude acessar e interagir com ferramentas externas em tempo real. Ao invés de copiar e colar informações entre sistemas, você autoriza o Claude a ler e agir diretamente nessas plataformas.

Ferramentas disponíveis
FerramentaO que o Claude consegue fazer
GmailLer e-mails, resumir threads, identificar pendências, rascunhar respostas
Google DriveBuscar documentos, resumir conteúdo, extrair informações de arquivos
Google CalendarConsultar agenda, listar reuniões, identificar conflitos de horário
NotionBuscar páginas, resumir notas, criar e atualizar conteúdo
SlackLer mensagens, resumir canais, encontrar informações em histórico
Como conectar
1Acesse Configurações → Connectors
2Selecione a ferramenta e autorize o acesso
3Defina o escopo — você controla o que o Claude pode ou não ver
Aplicação no Micro-SaaS

Conectar o Google Drive permite ao Claude acessar diretamente transcrições de entrevistas, pesquisas e planilhas do projeto — sem precisar copiar o conteúdo para o chat toda vez. Conectar o Notion permite que ele atualize seu board de tarefas enquanto você conversa.

⚠️ Atenção: os Connectors têm acesso real às suas contas. Revise cuidadosamente quais permissões você concede e nunca autorize escopos mais amplos do que o necessário para a tarefa.
02. Cowork
O Claude que trabalha enquanto você não está olhando

O Cowork é um produto separado da interface de chat — ele roda no desktop e permite que o Claude acesse arquivos locais do seu computador, execute tarefas em múltiplos passos de forma autônoma e entregue resultados diretamente na sua pasta de trabalho.

Chat vs. Cowork
ChatCowork
Você conversa, o Claude respondeVocê delega uma tarefa, o Claude executa
Contexto limitado à conversaAcesso real a arquivos e pastas do seu computador
Resultado entregue no chatResultado entregue como arquivo na sua pasta
Ideal para pensar juntoIdeal para processar, organizar e produzir
Casos de uso práticos
1Processar todas as transcrições de entrevistas de uma pasta e gerar um relatório consolidado
2Organizar arquivos do projeto por categoria e renomear com padrão consistente
3Ler uma planilha de dados e gerar um resumo executivo em PDF
4Criar um relatório de status semanal com base nos arquivos modificados recentemente
Como funciona

Você concede ao Cowork acesso a uma pasta específica do seu computador — nunca ao sistema inteiro. Dentro dessa pasta, o Claude lê os arquivos, executa o fluxo de trabalho descrito e salva os resultados. Você define o escopo, ele executa.

💡 Dica: crie uma pasta dedicada ao seu projeto Micro-SaaS no computador antes de usar o Cowork. Mantenha nela: transcrições de entrevistas, pesquisas, documentos do projeto e arquivos de código. Isso limita o acesso ao que é relevante e mantém tudo organizado.
⚠️ Preview: o Cowork está em preview — algumas funcionalidades ainda são instáveis. Comece com tarefas de leitura e geração antes de testar tarefas que modificam ou reorganizam arquivos existentes.
03. Claude Code
Construindo o Micro-SaaS em linguagem natural

O Claude Code é uma ferramenta de linha de comando que transforma o Claude num agente de desenvolvimento. Você descreve o que quer construir em linguagem natural — no terminal — e o Claude lê o código existente, escreve novos arquivos, executa comandos e gerencia o repositório.

O que o Claude Code consegue fazer
ScaffoldEstrutura inicial do projeto
Cria a estrutura de pastas, arquivos de configuração e a rota principal a partir de uma descrição em linguagem natural.
CódigoEscreve em qualquer linguagem
Python, JavaScript, TypeScript, SQL e outros. Lê o código existente antes de escrever, mantendo consistência com o que já foi criado.
DebugCorrige bugs e explica
Corrige bugs a partir de uma descrição do problema e explica em linguagem simples o que cada parte do código faz.
TestesCria testes automatizados
Escreve testes para o código existente e documenta funções e módulos.
Conceito essencial — CLAUDE.md

O CLAUDE.md é o arquivo de instruções do projeto para o Claude Code — equivalente às "instruções do projeto" que você escreveu no Project da Aula 2, mas para o ambiente de desenvolvimento. Fica na raiz do repositório e define: o que o projeto faz, como está estruturado, quais convenções de código seguir e quais comandos rodar para testar.

Instalação

O Claude Code requer Node.js instalado. Para instalar e iniciar:

Terminal — Instalaçãonpm install -g @anthropic-ai/claude-code
Terminal — Iniciar sessãoclaude
Fluxo básico de uso
1Abra o terminal e navegue até a pasta do projeto
2Execute claude para iniciar a sessão
3Descreva o que você quer construir em linguagem natural
4O Claude lê os arquivos existentes, escreve o código e explica o que fez
5Você testa, avalia e pede ajustes na mesma sessão
💡 Dica: você não precisa saber programar para usar o Claude Code — mas quanto mais você entender do código que ele gera, mais precisos serão os seus pedidos. Use a Aula 1 como base: itere, peça explicações, faça perguntas.
⚠️ Plano necessário: o Claude Code requer plano Pro ou Max. No plano gratuito, use o Chat com Artifacts de código como alternativa — menos integrado ao ambiente de desenvolvimento, mas funciona para criar e revisar código.
04. O Fluxo Completo
Como tudo se conecta no seu Micro-SaaS

Com todas as ferramentas que você aprendeu ao longo do curso, o fluxo de construção do seu Micro-SaaS com o Claude fica assim:

Fluxo completo do cursoAula 1 — Chat + Web Search └── Hipótese de problema documentada Aula 2 — Projects + Deep Research └── Roteiro de entrevistas + pesquisa de contexto Aula 3 — Artifacts + Skills └── Análise de entrevistas + proposta de valor definida Aula 4 — Connectors + Cowork + Claude Code └── Scaffold do MVP + automações + integrações ativas

A partir de agora, o Claude não é mais só uma ferramenta de pesquisa e escrita — ele é parte do time de desenvolvimento do seu projeto.

05. Demonstração ao Vivo
Acompanhe a tela e observe os resultados

O professor vai demonstrar ao vivo os quatro cenários desta aula. Acompanhe e observe como cada ferramenta funciona na prática.

1 Conectando o Google Drive. Conexão ao vivo em Configurações → Connectors, autorização de acesso e pedido ao Claude para localizar e resumir um documento do projeto. O Claude acessa o arquivo diretamente, sem upload manual.
2 Cowork processando arquivos. Abertura do Cowork, concessão de acesso a uma pasta com transcrições de entrevistas e instrução para processar todos os arquivos e gerar um relatório consolidado. Arquivo criado diretamente na pasta, sem intervenção manual.
3 Claude Code gerando o scaffold. Terminal aberto, pasta vazia, claude executado. Funcionalidade principal do Micro-SaaS descrita em linguagem natural. Estrutura do projeto gerada em tempo real — cada arquivo explicado.
4 Iterando sobre o código. A partir do scaffold, pedidos progressivos: rota de cadastro, modelo de banco de dados, explicação em linguagem simples. O Claude lê o código existente antes de escrever, mantendo consistência.
06. Atividade Prática
Sua vez. Cada passo leva de 5 a 10 minutos.
Passo 1 — Conecte pelo menos uma ferramenta

Acesse Configurações → Connectors e conecte a ferramenta mais útil para o seu fluxo. Sugestão: comece com Google Drive ou Notion, onde você já tem documentos do projeto. Depois teste:

Prompt — Testar ConnectorBusque no meu [Google Drive / Notion] um documento relacionado ao meu projeto de Micro-SaaS e me dê um resumo do conteúdo e quais são os próximos passos indicados nele.
Passo 2 — Defina a automação do Cowork

Escreva a instrução de automação que você vai querer rodar toda semana ao longo do semestre:

Template — Automação semanal (Cowork)Toda semana, acesse a pasta [nome da pasta] e: 1. Leia todos os arquivos novos ou modificados desde a última execução 2. Identifique: o que foi feito, o que está em andamento e o que está bloqueado 3. Gere um relatório de status em Markdown com: - Resumo executivo (3 linhas) - Lista de avanços da semana - Próximos passos priorizados - Dúvidas ou bloqueios identificados 4. Salve o relatório como "status_[data].md" na pasta /relatorios
Passo 3 — Gere o scaffold do seu Micro-SaaS

Abra o terminal, crie uma pasta para o projeto e execute claude. Se não tiver o Claude Code disponível, use o Chat e peça um Artifact de código.

Prompt — Scaffold do Micro-SaaSPreciso criar o scaffold inicial de um Micro-SaaS com as seguintes características: Problema que resolve: [descreva o problema] Usuário principal: [perfil do usuário] Funcionalidade core: [descreva a funcionalidade principal] Stack preferida: [ex: Python + FastAPI, ou Node.js + Express] Crie a estrutura inicial do projeto com: - Estrutura de pastas organizada - Arquivo principal com a rota mais importante funcionando - README com instruções de instalação e execução - Arquivo CLAUDE.md com as convenções do projeto
Passo 4 — Primeira iteração de desenvolvimento

Com o scaffold gerado, faça pelo menos dois pedidos adicionais:

Iteração 1 — Nova rotaAdicione uma rota para [funcionalidade secundária do seu Micro-SaaS]. Mantenha consistência com o código já existente.
Iteração 2 — Modelo de dadosCrie um modelo de banco de dados simples para armazenar [entidade principal]. Use [SQLite / PostgreSQL / MongoDB] e explique as decisões tomadas.
Iteração 3 — Entender o projetoExplique em linguagem simples como está estruturado o projeto que você criou. Para cada arquivo, descreva o que ele faz e por que está ali.
Passo 5 — Seu plano para o resto do semestre

Abra uma conversa no seu Project do Micro-SaaS e use o prompt abaixo para criar um plano concreto das próximas semanas:

Prompt — Plano do semestreCom base em tudo que discutimos neste projeto, crie um plano de uso do Claude para as próximas 8 semanas do semestre. Para cada semana, indique: - Em qual etapa do Micro-SaaS estarei (validação / priorização / construção / lançamento) - Quais ferramentas do Claude vou usar (Chat, Projects, Deep Research, Artifacts, Skills, Connectors, Cowork, Claude Code) - Qual o principal entregável da semana - Um prompt sugerido para começar a semana Formate como uma tabela Artifact.
