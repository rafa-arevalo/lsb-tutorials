Esta é a primeira aula do curso e o ponto de partida para tudo que vem a seguir. Antes de automatizar, pesquisar, construir e entregar, você precisa aprender a conversar bem com o Claude.

A maioria das pessoas que usa o Claude pela primeira vez trata ele como um motor de busca — faz perguntas curtas, recebe respostas genéricas e conclui que "a IA não é tão boa assim". O problema não é o Claude. É a forma de perguntar.

Nesta aula você vai entender como o Claude funciona, aprender a estrutura de um bom prompt e descobrir como a Memory faz com que o Claude te conheça melhor a cada conversa. Ao final, você vai sair com sua primeira hipótese de problema documentada.

Objetivos de aprendizagem
1Entender o que é o Claude e como ele se diferencia de outros assistentes de IA
2Formular prompts eficazes usando a estrutura: contexto + papel + tarefa + formato + restrições
3Iterar dentro de uma mesma conversa sem precisar recomeçar do zero
4Ativar e gerenciar a Memory do Claude para que ele aprenda suas preferências ao longo do semestre
01. O que é o Claude
E por que ele é diferente dos outros assistentes de IA

O Claude é um assistente de inteligência artificial desenvolvido pela Anthropic. Você conversa com ele em linguagem natural — como se estivesse mandando mensagem para alguém — e ele responde, analisa, escreve, calcula, explica e ajuda a pensar.

Três características fazem o Claude se destacar:

01Contexto longo
O Claude consegue ler e processar documentos inteiros numa única conversa — contratos, artigos, transcrições de entrevistas, planilhas. Você não precisa fragmentar o conteúdo.
02Raciocínio estruturado
O Claude consegue raciocinar em etapas, comparar opções, identificar inconsistências e explicar o porquê das suas conclusões — não apenas responder superficialmente.
03Sem memória entre sessões (por padrão)
Cada conversa começa do zero, a menos que você ative a Memory ou use Projects. Por enquanto, assuma que o Claude "esquece tudo" quando você fecha a janela.
02. Anatomia de um Prompt
A estrutura que transforma respostas genéricas em respostas úteis

Prompt é o nome que damos à mensagem que você envia ao Claude. Existe uma diferença enorme entre um prompt fraco e um prompt forte.

Prompt fraco vs. Prompt forte
Prompt fraco ✗Me ajuda com meu projeto.
Prompt forte ✓Você é um especialista em produto para startups B2B de saúde. Estou desenvolvendo um Micro-SaaS para clínicas odontológicas que resolve o problema de perda de pacientes por falta de follow-up pós-consulta. Me ajude a formular uma hipótese de problema clara em 3 frases, no formato: [Perfil] tem dificuldade em [problema] o que causa [consequência].
Os 5 ingredientes do prompt eficaz
IngredienteO que éExemplo
ContextoQuem você é e qual a situação"Estou desenvolvendo um Micro-SaaS para clínicas odontológicas."
PapelQual especialista o Claude deve ser"Aja como especialista em produto para startups de saúde."
TarefaO que exatamente você quer"Formule uma hipótese de problema clara."
FormatoComo quer receber a resposta"No formato: [Perfil] + [problema] + [consequência]."
RestriçõesO que ele deve evitar"Não mencione soluções tecnológicas. Foque nas dores humanas."
💡 Dica: você não precisa usar todos os cinco ingredientes sempre. Mas quanto mais contexto você fornecer, melhor tende a ser o resultado.
03. Iteração
O segredo que a maioria ignora

Você pode continuar a conversa. Não precisa ser perfeito na primeira mensagem.

O fluxo natural
1Você manda um prompt inicial
2O Claude responde
3Você avalia e pede ajustes na mesma conversa
4O Claude melhora levando em conta tudo que já foi dito
Mensagens de iteração que funcionam bem
Exemplos prontos para usarEstá bom, mas quero um tom mais informal. Reduz para metade do tamanho. Reescreva focando nas consequências financeiras. Gere mais 2 variações com perfis de usuário diferentes. Antes de continuar, me faça 3 perguntas para entender melhor meu projeto.
04. Memory
O Claude que aprende com você

A Memory permite ao Claude derivar preferências e contexto das suas conversas e carregá-los automaticamente nas sessões seguintes.

O que o Claude lembra

Preferências de comunicação, informações sobre seus projetos ativos, restrições que você mencionou, decisões já tomadas.

O que você pode fazer
1Visualizar tudo em Configurações → Memória
2Editar uma memória incorreta ou desatualizada
3Deletar memórias específicas ou apagar tudo
4Adicionar manualmente informações importantes do seu projeto
💡 Dica: logo após esta aula, acesse Configurações → Memória e adicione as informações do seu projeto para que o Claude as use em todas as conversas seguintes.
05. Demonstração ao Vivo
Todos os prompts prontos para copiar — acompanhe e depois repita
📋 Nota: esta seção contém o passo a passo completo da prática ao vivo, incluindo todos os prompts e dados sintéticos necessários. O professor executa no projetor — os alunos acompanham e depois repetem na atividade prática.
Demo 1Prompt fraco vs. prompt forte10 min

Objetivo: mostrar visivelmente a diferença de qualidade entre um prompt genérico e um estruturado.

Passo 1 — Execute o prompt fraco

Abra o Claude.ai numa nova conversa. Copie e envie:

Prompt fraco ✗Me ajuda com meu projeto de odontologia.
🎙️ Fala para a turma: "Observem a resposta. O Claude não tem contexto nenhum — ele não sabe quem somos, o que queremos, nem para que serve. O resultado é previsível: vago e inútil."
Passo 2 — Abra uma nova conversa e execute o prompt forte
Prompt forte ✓Você é um especialista em desenvolvimento de produto para startups de saúde. Contexto: estou desenvolvendo um Micro-SaaS para clínicas odontológicas de pequeno porte (1 a 3 dentistas) que perdem pacientes porque não fazem follow-up após as consultas. Tarefa: me ajude a formular uma hipótese de problema clara. Formato: use a estrutura — "[Perfil do usuário] tem dificuldade em [problema específico] o que causa [consequência mensurável para o negócio]." Gere 2 variações com perfis de usuário ligeiramente diferentes. Restrições: não mencione soluções tecnológicas ainda. Foque exclusivamente nas dores humanas e operacionais.
🎙️ Fala para a turma: "Notem como o Claude agora entrega algo específico, estruturado e diretamente útil. A diferença não está no Claude — está na qualidade do pedido."
Passo 3 — Compare as duas respostas

Role a tela para mostrar as duas janelas abertas. Peça à turma para identificar as diferenças.

Demo 2Iteração em conversa10 min

Objetivo: mostrar que é possível refinar progressivamente sem recomeçar do zero.

Passo 1 — Continue na conversa da Demo 1 que gerou as duas variações
Gostei da segunda variação. Agora reescreva ela tornando a consequência mais específica — use números estimados para tornar o impacto mais tangível. Exemplo: "o que causa perda estimada de R$ X por mês em receita de retorno."
Passo 2 — Após a resposta, envie:
Ótimo. Agora reformule em linguagem mais simples, como se eu fosse apresentar para um colega que não tem familiaridade com startups. Máximo de 2 frases.
Passo 3 — Após a resposta, envie:
Perfeito. Antes de continuarmos, me faça 3 perguntas que vão me ajudar a refinar ainda mais essa hipótese de problema.
🎙️ Fala para a turma: "Em três mensagens saímos de uma hipótese genérica para algo bem afinado — e o Claude manteve o contexto de tudo que foi dito antes, sem precisar repetir nada. Esse é o poder da iteração."
Demo 3Claude como parceiro de raciocínio10 min

Objetivo: mostrar o Claude ajudando a pensar, não apenas executando.

Passo 1 — Abra uma nova conversa e envie:
Tenho uma ideia para um Micro-SaaS voltado para clínicas odontológicas. Minha percepção é que as clínicas perdem muitos pacientes porque não fazem acompanhamento após as consultas. Antes de me ajudar a estruturar qualquer coisa, me faça as perguntas que você precisaria responder para avaliar se esse é realmente um problema relevante e viável de resolver com tecnologia.
Passo 2 — Responda as perguntas do Claude com estes dados:
Dados sintéticos para a demoRespondendo suas perguntas: 1. Sobre frequência do problema: dentistas relatam que 40% a 60% dos pacientes não retornam após a primeira consulta sem um contato ativo da clínica. 2. Sobre como lidam hoje: a maioria usa WhatsApp manual ou simplesmente não faz follow-up — fica na memória da recepcionista. 3. Sobre impacto financeiro: uma consulta de retorno vale entre R$ 200 e R$ 400. Uma clínica pequena perde cerca de 20 pacientes por mês por falta de follow-up. Isso representa entre R$ 4.000 e R$ 8.000 mensais em receita não realizada. 4. Sobre soluções existentes: há CRMs genéricos (Salesforce, Pipedrive) mas são caros e complexos para clínicas pequenas que não têm equipe de TI. Softwares odontológicos focam em prontuário, não em relacionamento.
Passo 3 — Após a resposta, envie:
Com base nas minhas respostas, formule a hipótese de problema mais forte possível para o meu Micro-SaaS. Em seguida, avalie: essa hipótese é suficientemente específica para guiar entrevistas de validação com dentistas?
🎙️ Fala para a turma: "Notem que não precisei ter a resposta perfeita de antemão. O Claude me ajudou a chegar lá através de perguntas — isso é usar IA como parceiro de raciocínio."
Demo 4Configurando a Memory5 min

Objetivo: mostrar como salvar contexto para uso automático nas próximas sessões.

Passo 1 — Acesse Configurações → Memória

Mostre o painel para a turma. Se houver memórias já geradas automaticamente, comente brevemente.

Passo 2 — Clique em "Adicionar memória" e insira:
Memória do projeto — exemploProjeto atual: Micro-SaaS para clínicas odontológicas de pequeno porte. Problema: falta de follow-up com pacientes após consultas, causando perda de receita de retorno. Contexto: estudante de AI for Business, fase de validação de problema. Preferência: respostas diretas, objetivas e com exemplos práticos.
Passo 3 — Abra uma nova conversa e envie apenas:
Com base no que você sabe sobre meu projeto, qual é o perfil de usuário mais importante que eu deveria entrevistar primeiro para validar o problema?
🎙️ Fala para a turma: "O Claude usou o contexto da Memory sem eu precisar repetir nada. A partir de agora, toda nova conversa já começa com esse contexto carregado automaticamente."
06. Atividade Prática
Sua vez — cada passo leva de 5 a 10 minutos
Passo 1 — Seu primeiro prompt fraco

Abra o Claude e escreva, da forma mais natural possível, uma mensagem sobre o problema que o seu Micro-SaaS resolve. Não tente ser perfeito — mande como se fosse uma mensagem de WhatsApp.

Passo 2 — Reescreva com estrutura

Reescreva o mesmo prompt usando: contexto + papel + tarefa + formato + restrições. Mande numa nova conversa e compare os dois resultados lado a lado.

Passo 3 — Peça ao Claude que te faça perguntas
Prompt — adapte para o seu segmentoTenho uma ideia de Micro-SaaS para [descreva seu segmento]. Minha percepção inicial é que [descreva o problema que você identificou]. Antes de me ajudar a estruturar a hipótese, me faça 3 perguntas que vão me ajudar a entender se esse problema é realmente relevante.

Responda as perguntas e peça ao Claude para formular a hipótese final.

Passo 4 — Configure sua Memory
Template de memória — adapte para o seu projetoProjeto atual: Micro-SaaS para [seu segmento]. Problema: [descreva o problema em 1 frase]. Contexto: estudante de AI for Business, fase de [fase atual]. Preferência: [seu estilo preferido de resposta].

Acesse Configurações → Memória, clique em "Adicionar memória" e cole o texto adaptado.
